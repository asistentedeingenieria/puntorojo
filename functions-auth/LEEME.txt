PUNTO ROJO — Cloud Functions
================================================================================

Dos funciones (se despliegan juntas, un solo comando):

  1) deleteAuthOnUserDoc
     Cuando borrás un usuario en la app (GESTIÓN DE USUARIOS → X), borra TAMBIÉN
     su cuenta de Firebase Authentication (el "registro de Firebase") y limpia
     sus notificaciones/tokens. Automático.

  2) sendPushOnNotification
     Cuando se crea una notificación para un usuario (campanita), le manda un
     PUSH al celular AUNQUE TENGA LA APP CERRADA. Limpia tokens vencidos.

Requisitos:
  - Node.js instalado (ya lo tenés).
  - Plan Blaze activo (ya lo tenés).

PASOS (una sola vez) — abrí PowerShell EN ESTA CARPETA (puntorojo-functions):
--------------------------------------------------------------------------------
  1) Instalar la herramienta de Firebase (si no la tenés):
        npm install -g firebase-tools

  2) Iniciar sesión (abre el navegador; entrá con la cuenta de Google dueña del
     proyecto punto-rojo-3fcf1):
        firebase login

  3) Instalar dependencias:
        cd functions
        npm install
        cd ..

  4) Desplegar las dos funciones:
        firebase deploy --only functions

     - La 1ª vez puede pedir habilitar APIs (Cloud Functions, Cloud Build,
       Artifact Registry) -> aceptá. Tarda 2-5 minutos.
     - Al terminar: "Deploy complete!" y lista deleteAuthOnUserDoc + sendPushOnNotification.

PRUEBAS
--------------------------------------------------------------------------------
  Borrado de Auth:
    Borrá un usuario de prueba en la app -> en Firebase Console -> Authentication
    la cuenta desaparece en segundos. (Functions -> Logs: "Cuenta de Auth eliminada").

  Push al celular (app cerrada):
    - Cada persona debe haber ACEPTADO las notificaciones la 1ª vez que abrió la
      app (el navegador/PWA pide permiso). Sin eso no hay token y no llega push.
    - En iPhone: la PWA debe estar "Agregada a pantalla de inicio" para recibir push.
    - Prueba: que alguien haga un movimiento que te genere notificación, con tu app
      CERRADA -> te debe sonar el push. (Functions -> Logs: "Push a <uid> ...").

Notas:
  - No borra al admin maestro ni tu propia cuenta (la app lo impide).
  - Si una cuenta de Auth ya no existía, la función lo ignora sin error.
